# SDU-IPTV-PRO

本项目汇集了山东联通、山东移动及山东电信三大运营商的 IPTV 内网直播源，并针对各地市用户收视习惯对频道顺序进行了本地化排列，方便不同地区用户快速使用。

## 📁 文件说明

> **说明**：文件名前缀 `SDU` / `SDM` / `SDT` 分别对应 **山东联通**、**山东移动** 与 **山东电信**。

## 📶 联通单播源特别说明

### 单播

山东联通单播采用 **RTP over UDP** 传输方式。处于 NAT 内网环境下的设备通常无法直接接收 RTP 数据包。如需正常收看单播节目，请确保满足以下任一条件：

- 拥有公网 IPv4 地址，并通过 PPPoE 接口接入互联网（山东联通宽带默认下发公网 IP；若未获取到公网 IP，可尝试在宽带账号后添加 `@e` 后缀并重新拨号）。
- 若不具备公网 IPv4，可通过部署 `rtsproxy` 等 NAT 穿透工具实现。
- 直接从 IPTV 专用接口访问（仅支持宽带所在地市的单播源）。

### 组播

- 推荐使用 `rtp2httpd` 等工具将组播流转为 HTTP 流，同时支持 FCC（快速频道切换），起播与换台体验接近单播。
- 播放终端也可直接接收并播放组播数据流。

> 参考自：[plsy1/iptv](https://github.com/plsy1/iptv)

## 🛜 联通组播源第三段及 FCC 说明

山东联通组播平台分为 **中兴** 与 **华为** 两套体系，FCC 服务器地址不通用。

- **中兴平台**：各地市 FCC 地址统一为 `124.132.240.66:15970`，通过负载均衡跳转至本地服务器。
- **华为平台**：各地市 FCC 地址各不相同，需自行抓包获取。若您有相关信息，欢迎通过 Issue 提交补充。

组播地址格式示例：  
`rtp://239.253.xxx.77:8000?fcc=xx.xx.xx.xx:xxxx`
